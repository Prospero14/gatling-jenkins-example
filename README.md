# Пример запуска Gatling через Jenkins

Добро пожаловать в репозиторий с примером автоматизации нагрузочного тестирования Gatling с помощью Jenkins.

Здесь собраны все необходимые файлы, чтобы Jenkins сам скачивал проект, давал тебе удобные менюшки для выбора сервера и симуляции, а потом запускал тесты.

## Из чего состоит проект:
1. **`Jenkinsfile`** — Это самый главный файл. Это "инструкция" для Jenkins. В нем написано, что нужно спросить у человека (какой сервер, какая симуляция) и какую команду выполнить.
2. **`pom.xml`** — Это файл настроек сборщика Maven. Он говорит Maven, что это проект Gatling, скачивает сам Gatling нужной версии и плагины для него.
3. **`src/test/java/simulations/`** — В этой папке лежат сами скрипты тестирования (симуляции). Для примера здесь лежит `Debug.java`. Именно сюда ты потом положишь свои рабочие скрипты.

---

## Как это работает в Jenkins?

Когда ты настроишь задачу (Job) в Jenkins и первый раз нажмешь **Собрать (Build)**, Jenkins прочитает файл `Jenkinsfile`.

С этого момента кнопка "Собрать" изменится на **Собрать с параметрами (Build with Parameters)**. При нажатии на нее Jenkins покажет тебе два простых выпадающих списка:
1. На каком сервере запускать (твои сервера, где установлен генератор)
2. Какую симуляцию запустить

После того, как ты выберешь нужные значения и нажмешь "Запуск", Jenkins:
- Подключится к выбранному серверу.
- Передаст ему команду на запуск нужной симуляции (`mvn gatling:test...`).
- Передаст настройки памяти (`MAVEN_OPTS`).
- Сохранит красивые HTML-отчеты о результатах теста.

## Как добавить свои сервера

Открой файл `Jenkinsfile`. Там ты увидишь блок `parameters`.
В параметре `TARGET_SERVER` есть список `choices`. Просто перечисли там имена (или IP адреса) серверов, на которых у тебя стоят воркеры/генераторы.

*Важно:* Чтобы Jenkins мог запустить тест на другом сервере, у него должен быть настроен доступ к этому серверу (SSH или Jenkins agent), и имя в списке `choices` должно соответствовать имени агента в (Manage Jenkins -> Nodes).
